# 실행 방법 (How to Run) — HardNeg-GRPO v6

팀 **pk42ac** · 최종 모델 **HardNeg-GRPO (v6)** · base Qwen3-VL-8B-Instruct (Apache-2.0)
가중치(공개): https://huggingface.co/psh3333/dacon-skku-bias-vlm-v6 · 코드: https://github.com/Sungho-pk42ac/dacon-2026-skku-multimodal-ai-bias-challenge

> **요약:** Private Score(실이미지 0.9628) 복원은 **공개 가중치를 받아 추론 1회**면 됩니다(아래 1·2절). 처음부터 학습 재현은 3절. 모든 명령은 RTX A6000에서 무에러 실행을 실측 검증했습니다(`REPRODUCIBILITY_검증.md`).

명령은 **번들 루트(=`code/`가 보이는 디렉터리)** 에서 실행합니다. 데이터는 `./data/`, 산출 모델은 `./outputs/`에 생깁니다.

## 2. Private Score 복원 — 공식 경로 (실이미지 v6 = 0.9628)

공개 가중치를 받아 **추론 1회**. 재학습 불필요.

```bash
python code/inference/make_submission.py \
    --model psh3333/dacon-skku-bias-vlm-v6 \
    --test_csv data/test.csv --image_root data/test/images \
    --sample_submission data/sample_submission.csv \
    --max_pixels 262144 \
    --out submission_v6_final.csv
```

- `--max_pixels 262144`(512²) = 채점 시와 동일 설정(DACON 0.5s/샘플 시간한도 내).
- greedy(`do_sample=False`)라 동일 입력이면 출력이 비트단위 동일 → 결정론적 복원.
- `make_submission.py`는 실제 테스트 이미지를 요구합니다. 정상 시 `real_image_loaded_count==전량`, `placeholder_fallback_count==0`(아니면 자동 INVALID/exit).

> ※ 텍스트집중(중립 회색 이미지) Public 1.0은 **ablation 관측**일 뿐 제출/평가 대상이 아닙니다. 관측 재현은 `--image_root <빈경로> --allow_placeholder`(파일 부재 시 336² 회색 자동생성).

## 3. 처음부터 전체 재현 (데이터 → SFT → 하드네거티브 채굴 → GRPO → 추론)

> 학습 코드는 `report_to="wandb"`가 하드코딩이라 **`export WANDB_MODE=disabled`** 를 먼저 설정하세요(로그 비활성, 로그인 불필요).

```bash
# [추론·데이터 env] 1) 외부데이터(공개 BBQ) → 클린셋 + val_ids(SSOT)  — 누출제거·옵션셔플·단일토큰
python code/data_build/make_bbq_clean.py --out_dir data
#   → data/bbq_v4_train.json, data/val_ids.json, data/placeholder.jpg 생성 (seed 42 결정론)

# [학습 env] 2) v4 = 단일토큰 SFT (Unsloth FastVisionModel)
source unsloth_env/bin/activate; export WANDB_MODE=disabled
python code/train/unsloth/sft_unsloth.py \
    --model Qwen/Qwen3-VL-8B-Instruct \
    --train data/bbq_v4_train.json --out outputs/merged_v4 --epochs 3 --rank 32

# [추론·데이터 env] 3) 하드네거티브 풀 채굴 (v4가 틀린 공개 QA 문항 + BBQ 앵커)
python code/data_build/mine_hard.py \
    --base outputs/merged_v4 --out hardpool.json --val_ids data/val_ids.json --per 500

# [학습 env] 4) v6 = 하드네거티브 GRPO (vLLM fast_inference, 최종 모델)
source unsloth_env/bin/activate; export WANDB_MODE=disabled
python code/train/unsloth/grpo_hard_v6.py \
    --base outputs/merged_v4 --out outputs/merged_v6 \
    --hardpool hardpool.json --steps 400 --rank 32 --num_gen 4

# [추론·데이터 env] 5) 제출 CSV 생성 (실이미지 0.9628)
python code/inference/make_submission.py --model outputs/merged_v6 \
    --test_csv data/test.csv --image_root data/test/images \
    --sample_submission data/sample_submission.csv --max_pixels 262144 \
    --out submission_v6_final.csv
```

### 빠른 동작 확인(스모크) — 무에러만 보고 싶을 때
- SFT: `... sft_unsloth.py ... --max_steps 15`  (epochs 대신 15스텝)
- GRPO: `... grpo_hard_v6.py ... --steps 5 --max_items 64`
