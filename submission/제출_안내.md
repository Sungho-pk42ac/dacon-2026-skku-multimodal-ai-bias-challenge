## 1. 최종 모델

- **HardNeg-GRPO (v6)** = Qwen3-VL-8B-Instruct(base) + 단일토큰 SFT + 하드네거티브 GRPO
- **가중치(HuggingFace, 공개):** https://huggingface.co/psh3333/dacon-skku-bias-vlm-v6
- **코드 저장소(GitHub, 공개):** https://github.com/Sungho-pk42ac/dacon-2026-skku-multimodal-ai-bias-challenge — 전체 학습·추론·실험 코드(제출 zip의 `code/`와 동일 스냅샷, 태그 `v6-submission`)
- **추론 결정성:** greedy 디코딩(`do_sample=False`) → 동일 모델·동일 입력이면 출력이 비트단위 동일 → **결정론적 복원** 보장.

## 2. Private Score 복원 — 공식 제출본 = **실이미지 v6 (0.9628)**

**2차 평가·제출 대상은 실이미지 멀티모달 추론입니다** (`docs/FINAL_MODEL_SELECTION.md` 확정). 2차 평가는 운영진이 제출 코드·모델로 **비공개(Hidden) 데이터셋에서 실이미지 성능을 재측정**하는 방식이므로, 복원·평가의 권위 경로는 아래 **단일 실이미지 명령**입니다.

```bash
# 0) 의존성 (추론만) — torchvision 포함 필수(Qwen3-VL 프로세서가 요구)
pip install -r code/requirements.txt

# 공식 복원 — 실이미지 멀티모달 (채점본 submission_v6_final.csv = 0.9628 재현)
#   DACON 0.5s/샘플 시간한도 내 추론을 위해 --max_pixels 262144(512^2) 사용(채점 시와 동일).
python code/inference/make_submission.py \
    --model psh3333/dacon-skku-bias-vlm-v6 \
    --test_csv data/test.csv --image_root data/test/images \
    --sample_submission data/sample_submission.csv \
    --max_pixels 262144 \
    --out submission_v6_final.csv
# → make_submission은 실제 테스트 이미지를 요구. real_image_loaded_count==전량 &
#    placeholder_fallback_count==0 이어야 정상(placeholder 발생 시 자동 INVALID/exit).
```

> **※ 텍스트집중(중립 회색 이미지) Public 1.0은 제출·평가 대상이 아닙니다.** 동일 v6에 실이미지 대신 중립 이미지를 넣었을 때 Public 리더보드에서 관측된 **ablation 현상**일 뿐이며, 2차 Hidden 평가(실이미지 멀티모달)와 무관합니다. 관측 재현이 필요하면 `--image_root <빈경로> --allow_placeholder`(파일부재 시 회색 자동생성)로 가능하나 **채점과 무관**합니다.

> 처음부터 재학습하는 전체 절차(SFT→하드네거티브 채굴→GRPO)는 `README_제출.md` 3절 참조.
> 학습 재현은 `code/requirements.txt`의 [학습] 섹션(Unsloth+vLLM, 별도 venv) 설치가 필요합니다.

## 4. 외부 데이터 (요건 1-②)

모두 **독립 공개 QA 데이터셋**이며, DACON 평가셋에서 파생하지 않았습니다.

| 데이터셋 | 용도 | 출처(HuggingFace) |
|---|---|---|
| BBQ (`walledai/BBQ`) | 편향/기권 문항 | 공개 |
| SIQA, CommonsenseQA, OpenBookQA, ARC | 일반추론 하드네거티브 | 공개 |

- 생성·정제 스크립트: `code/data_build/*.py` (대표: `make_bbq_clean.py`, `mine_hard.py`) — **seed 42 결정론적 재생성**.
- 누출 방지: sha1 `sample_id`로 train/val/eval 전역 배제 + assert (`make_bbq_clean.py`).
- ※ 생성 산출물 실파일(`bbq_v4_train.json`, `hardpool.json`, `val_ids.json`)은 위 스크립트로 동일 재생성됩니다(전부 공개 데이터 기반). 운영진 요청 시 실파일도 추가 제공 가능합니다.
