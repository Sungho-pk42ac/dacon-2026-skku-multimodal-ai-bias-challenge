# Robustness-Aware GRPO for Multimodal Bias Mitigation: A Negative-Result-Informed Study

> DeepSeek-R1(2501.12948) 논문 구조를 따른 v8B 논문 초안. base: Qwen3-VL-8B-Instruct (Apache-2.0).
> 모든 정량치는 동일 held-out 공개셋 v8_eval(900)에서 측정. DACON 평가셋 미사용.
> **정직성 원칙**: R1과 분명히 다른 점(단일토큰=비CoT, BBQ 포화, 노이즈수준 향상)은 그대로 명시한다.

---

## Abstract
멀티모달 BBQ 스타일 3지선다 편향 문항에서 사회적 고정관념에 휘둘리지 않고 근거가 부족하면
'판단 불가'를 택하는 8B VLM을 구축한다. 우리는 **콜드스타트 SFT 없이 GRPO만으로**(R1-Zero의 순수 RL과
동형) 일반화를 보존할 수 있음을 보이고, **콜드스타트 SFT가 좁은 분포에서 파국적 망각을 일으킴**을
ablation으로 규명한다(음성결과). 나아가 **강건성 인지 GRPO**(옵션순서 일관성·소스정규화 보상 +
셔플짝 증강)로 옵션순서 강건성 회귀를 회복한다(v8B). R1과 달리 본 과제는 단일토큰 출력이라 추론
체인·"aha moment"는 없으며, BBQ 정확도는 포화(1.0)되어 정량 향상은 노이즈 수준임을 정직히 보고한다.
주된 기여는 점수가 아니라 **무엇이 일반화를 살리고 죽이는가**에 대한 ablation 기반 분석이다.

## 1. Introduction & Contributions
- **과제**: 이미지+context+question+3선택지 → 정답 인덱스(0/1/2). 한 선택지는 기권('cannot be determined'). ambiguous는 기권이 정답, disambiguated는 본문 근거가 정답.
- **기여**
  1. **GRPO-only가 일반화를 보존**(R1-Zero의 "순수 RL로 능력 창발"과 동형) — 콜드스타트 불필요.
  2. **콜드스타트 SFT의 파국적 망각을 ablation으로 규명**(음성결과). R1은 *수천 개 다양한 CoT*로 콜드스타트에 성공했으나, 좁은(BBQ 100%)·암기 유도 콜드스타트는 일반화를 붕괴시킴 → "콜드스타트 자체가 아니라 데이터 다양성·비암기성이 관건".
  3. **강건성 인지 GRPO**: 옵션셔플 일관성·소스정규화 보상 + 셔플짝 데이터증강으로 v8A의 강건성 회귀 회복.
  4. **규칙기반 outcome 보상만 사용**(신경망 보상모델 X) — R1의 반-PRM 논거와 동일.

## 2. GRPO-only Bias Mitigation (v4→v6) — R1-Zero에 대응
*(R1-Zero = SFT 없이 순수 RL. 우리는 가벼운 단일토큰 SFT(v4) 위에 GRPO-only를 적용.)*

### 2.1 GRPO 알고리즘
critic 없이 한 프롬프트에 G개 출력 샘플링 → 그룹 정규화 advantage `A=(r−mean)/std` + KL 규제 (Shao et al. 2024, DeepSeekMath).

### 2.2 규칙기반 보상 (신경망 보상모델 미사용)
- **정답 보상**: 정답 인덱스 일치 +2 / 오답 −1 (자동검증, 결정론적).
- **형식 보상**: 단일토큰 0/1/2 유효 +0.2 / 위반 −0.5.
- R1과 동일하게 신경망 보상모델을 피함(reward hacking·비용). (R1 §Unsuccessful Attempts의 PRM 기각과 동일 논거.)

### 2.3 템플릿 — R1과의 핵심 차이
R1은 `<think>` 체인을 유도하지만, **본 과제는 단일토큰(0/1/2) 직답**이다. 따라서 추론 체인·테스트타임
연산 증가·"aha moment" 같은 창발은 **없다**(정직히 명시). 이는 편향 판단이 긴 추론보다 *근거 유무
판정*에 가깝고, 추론 도입이 오히려 편향을 키울 수 있다는 보고(2502.15361)와도 부합한다.

### 2.4 자기진화 대신 "포화"
R1-Zero는 AIME 15.6→71.0%로 성장하지만, 본 과제의 BBQ는 **이미 천장(1.0)**이라 GRPO가 추가로
끌어올릴 여지가 없다. v5(GRPO on BBQ 전체)는 보상 분산≈0 → **효과 0**. 향상은 v4 잠재능력 한계 내.

### 2.5 한계
- 포화 분포에선 GRPO 신호 부재(v5). v6(하드네거티브)는 v4와 동급·안정(일반화 보존).

## 3. v8B: Robustness Pipeline — R1 다단계 파이프라인에 대응
*(R1 = 콜드스타트→추론RL→거부샘플링SFT→전시나리오RL. 우리는 콜드스타트를 의도적으로 제거하고,
데이터 큐레이션 + 강건성 보상으로 대체.)*

### 3.1 데이터 큐레이션 (R1의 콜드스타트 데이터 수집에 대응, 단 SFT 아님)
- 공개데이터 후보 1560(소스당 260, val_ids+v8_eval 제외)을 v4·v8A 두 모델로 채점(원본+셔플) + v8A G=8 동적샘플링(DAPO 오프라인 근사).
- 6 집중 카테고리: 셔플불일치/회귀/양쪽오답/사람선택오류/과기권/OOD이득 → 170 고유 + **셔플짝 = 340행**.
- BBQ는 v4·v8A가 완벽(1.0)이라 동적샘플링상 too-easy로 자동 제외 → 풀은 OOD 집중.

### 3.2 강건성 인지 보상 (6종, 개별 로깅)
1. answer(+2/−1) 2. **shuffle_consistency**(예측 옵션텍스트가 순서무관 정답텍스트와 일치 +1/−1)
3. abstain(amb→unk +0.5 / 위반 −0.5 / dis 과기권 −0.6) 4. **source_normalized**((정답?1:0)−v8A기준정확도[src], 소스별 zero-centered)
5. format(+0.2/−0.5) 6. length_penalty. — 셔플짝 풀과 결합되어 **옵션순서 불변**을 직접 최적화.

### 3.3 학습 설정
base merged_v4, LoRA rank 16(attention-only, vision frozen), lr 1e-6, 200 steps, num_gen 8, temp 0.85,
단일토큰(max_completion 4), bf16, wall 28분. **콜드스타트 없음**(§5 음성결과 근거).

## 4. 8B + RLVR의 한계 — R1의 "Distillation vs RL"에 대응
R1은 "작은 모델엔 distillation >> 직접 RL"을 보였다(R1-Zero-Qwen-32B ~50% vs Distill 72.6% AIME).
본 연구도 **8B에 GRPO 직접 적용의 향상폭은 노이즈 수준**임을 관측 — RLVR은 base 잠재능력을 끌어낼 뿐,
한계를 넘지 못한다는 동일한 결론. (v4≈v6≈v8A≈v8B, BBQ 포화 + OOD ±수문항.)

> **핵심(개선폭 입증)**: zero-shot VLM은 *ambiguous 문항에서 고정관념 인물을 선택*한다 —
> LLaVA-OV는 amb에서 사람선택오류 **0.8267**(83% 편향), base도 0.18. 파인튜닝 후 **0.0**(s_AMB 0.0867/0.2667→0.0).
> 즉 balanced 1.0은 단순 포화가 아니라 **편향제거의 직접 증거**. 또한 base는 단일토큰 형식 유효율 0.0(자유텍스트),
> 파인튜닝이 형식 준수까지 확보. (v4 참고치: OOD 0.8033, 셔플 0.9478 — 별도 prior eval.)

> v6와 v8B는 독립셋에서도 실질 동급(v6 미세우위) → 제출모델은 v6 선택(§FINAL_MODEL_SELECTION).

## 6. Discussion
### 6.1 콜드스타트 vs GRPO-only
ablation: v7은 GRPO 이전(콜드스타트 SFT만)에 이미 OOD 붕괴 → **붕괴 원인은 콜드스타트 SFT**(GRPO 무죄).
Chu et al. 2025("SFT Memorizes, RL Generalizes") 예측과 일치. R1의 콜드스타트 성공과의 차이는 §1.2(데이터 다양성).
### 6.2 이미지 분포 발견 (신규)
모델이 placeholder(336²)로 학습돼, **실제 테스트 이미지(고해상도) 제출 시 점수 1.0→0.96**으로 하락.
멀티모달 평가에서 *학습/추론 이미지 분포 불일치*가 점수를 좌우함을 보임. (본 모델은 텍스트 위주 — §7 한계.)

## 7. Unsuccessful Attempts — R1의 동명 섹션에 대응
- **v5 (포화 분포 GRPO)**: BBQ val 포화 → 보상 분산≈0 → 향상 0.
- **v7 (콜드스타트 SFT + 추론 GRPO)**: BBQ 100% 콜드스타트 + 정답박은 추론템플릿 + rank64 →
  amb 0.707/OOD 0.673/셔플 0.794/사람오류 0.293/s_AMB 음수 **전방위 붕괴**. ablation으로 콜드스타트가 범인 확정.
- **v8A (강건성 GRPO 1차)**: OOD 최고(0.8117) 얻었으나 옵션셔플 일관성 0.9411로 **회귀**(trade-off). format 보상 죽은신호 버그.
- **신경망 보상모델 회피**: R1의 PRM/MCTS 기각과 동일하게, 규칙기반 outcome 보상만 사용(reward hacking·비용 회피).

## 8. Limitations & Future Work
- **멀티모달이나 이미지 의존도 낮음**(placeholder 학습) — 가장 큰 리뷰어 리스크. 향후 실이미지 소량 SFT로 진짜 멀티모달화 필요.
- 단일 시드(통계적 유의성 미확보) — 다중 시드 + 신뢰구간 필요.
- v8B 보상 컴포넌트 개별 ablation 미실시 — 각 보상 기여 입증 필요.
- ~~base(no-FT) 베이스라인 미측정~~ → **완료**(§5.1/5.2: base·LLaVA-OV-7B 측정, 개선폭 입증).
- ood2 외부검증은 v6/v8B/base 3종(+텍스트전용이라 image-invariance ablation만). v8A·real/shuffled-image ablation·VQA/GQA는 미실시(pod 자원 제약).
- 평가가 자체 공개데이터 분할 + 표준셋(MMLU/HellaSwag/ARC-C/WinoGrande) held-out. 표준 벤치마크 *공식 리더보드* 제출은 아님.
